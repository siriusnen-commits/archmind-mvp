# ArchMind Plan

- 목표: 실패 재현을 안정화하고 최소 변경으로 성공 상태를 만든다.
- 범위: 기능 수정, 테스트/검증, 결과 기록
- 제외: 대규모 리팩터링, 요구사항 외 신규 기능 추가

## 작업 단계
1. [S1] 현행 코드베이스 파악
   - 설명: 핵심 엔트리포인트와 실패 로그를 확인해 수정 대상 범위를 고정한다.
   - 검증: 재현 커맨드가 일관되게 실패/성공하는지 확인
1. [S2] 핵심 수정 구현
   - 설명: 우선순위 결함을 최소 변경으로 수정한다. (범위: backend/frontend 및 테스트)
   - 검증: 관련 테스트/검증 커맨드 통과
1. [S3] 회귀 검증
   - 설명: 수정 범위 외 기능에 영향이 없는지 회귀를 점검한다.
   - 검증: python -m pytest -q 또는 프로파일 커맨드 통과
1. [S4] 결과 정리
   - 설명: 변경 파일/리스크/남은 TODO를 기록하고 인수 기준 충족 여부를 판단한다.
   - 검증: acceptance 항목 모두 충족

## 테스트 전략
- 1차: 실패 재현 커맨드로 기준선 확인
- 2차: 수정 후 동일 커맨드 재실행으로 개선 여부 확인
- 3차: python -m pytest -q 또는 선택한 profile 검증
- 4차: 실패 원인/스택트레이스가 제거되었는지 로그 확인

## 리스크
- [R1] 수정 범위 확대: 불필요한 파일 변경으로 회귀 가능성 증가
  - 대응: 실패 지점 중심의 최소 수정 원칙 유지
- [R2] 환경 의존 실패: Node/Python 의존성 미설치로 검증 불가
  - 대응: SKIP 원인 명시 및 재실행 조건 문서화
